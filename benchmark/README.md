# 法律案件RAG测试题库

基于讯问笔录场景的测试题目，用于评测法律案件场景下的RAG系统质量。

## 📁 题库文件

```
benchmark/questions/
├── fact_exact.json       # 事实型题目 (10题)
├── evidence_set.json     # 证据集合型题目 (8题)  
└── conflict_gap.json    # 冲突/缺口型题目 (10题)
```

**总计：28道测试题**

## 📋 题型说明

### 1. fact_exact - 事实型
唯一答案的精确事实提取。
- 金额：42000元总收款、18000元退款、24000元欠款
- 日期：2020-06-05、2020-08-31、2020-11-24
- 个人信息：姓名、身份证号、电话
- 是/否问题：是否认罪认罚

### 2. evidence_set - 证据集合型
需要聚合多条证据的复杂问题。
- 承诺退款/已退款/仍欠款的证据
- 资金用途说明的证据
- 微信转账记录
- 联系和寻找过程

### 3. conflict_gap - 冲突/缺口型
测试系统是否会在信息缺失时编造内容。
- "忘记了/不详/不知道"的信息应拒答
- 禁止幻觉和猜测

## 📄 样本数据

基于 `SampleData/output.md` 讯问笔录：
- 被讯问人：陈明飞
- 涉案金额：42000元（收款）→ 18000元（退款）→ 24000元（欠款）
- 时间范围：2020年6月-11月

## 🎯 设计原则

参考 `docs/rag-讯问-询问笔录.md` 设计文档：

✅ **可核验**: 所有答案需提供证据引用  
✅ **零容错**: 金额/日期必须精确匹配  
✅ **冲突显式**: 不确定信息需明确说明  
✅ **召回优先**: 混合检索 + 重排  

## 💡 使用方式

将题目JSON集成到你的测试框架中，用于评估RAG系统在法律文档上的表现。

# MVP计划 (最小可行产品)

> 文档版本: v1.0
> 制定日期: 2026年1月12日
> 目标: 3个月内上线核心功能

---

## 1. MVP定位与目标

### 1.1 核心价值主张
定位: 刑事辩护律师的智能助手，快速理解案卷、发现辩护要点、提升办案效率

目标场景: 侦查阶段，律师收到案卷材料后快速分析

用户价值:
- 效率提升: 案卷分析从数周缩短到数小时
- 质量保障: 自动发现证据矛盾，避免遗漏
- 知识赋能: 提供质证要点和辩护建议

### 1.2 目标用户
1. 实习律师 (占比40%) - 最需要辅助
2. 助理律师 (占比35%) - 工作量最大
3. 独立律师 (占比25%) - 效率需求强

### 1.3 MVP目标
业务目标:
- 上线后3个月内: 20-50个种子用户
- 用户留存率: 30日留存 >40%
- 用户满意度: >4.0/5.0
- 付费转化率: >10%

产品目标:
- 覆盖5种常见案件类型
- 证据分类准确率 >85%
- 矛盾点识别准确率 >75%
- 案卷解析时间 <1分钟/100页

## 2. MVP核心功能 (5大功能)

### 功能1: 案卷智能导入与解析 (P0)
优先级: P0
描述: 支持PDF、图片(OCR)、JSON等格式导入，自动识别文书类型，提取元数据
用户故事: 作为律师，我需要快速导入案卷，以便开始分析
验收标准:
- 支持5种以上常见文书类型识别
- 识别准确率 >90%
- 单个案件导入时间 <1分钟

### 功能2: 案件概览自动生成 (P0)
优先级: P0
描述: 自动提取案件基本信息(案号、当事人、罪名、时间线)，生成案件摘要(300-500字)，程序节点可视化展示
用户故事: 作为律师，我需要快速了解案件基本情况，以便制定策略
验收标准:
- 案件基本信息提取准确率 >95%
- 摘要覆盖度 >90%
- 时间线完整性 >95%

### 功能3: 证据自动分类与汇总 (P0)
优先级: P0
描述: 自动分类证据类型(人证、物证、鉴定、辨认等)，每类证据数量统计，证据清单导出(Excel)
用户故事: 作为律师，我需要自动分类证据，以便系统管理
分类体系:
1. 犯罪嫌疑人供述: 讯问笔录
2. 被害人陈述: 询问笔录
3. 证人证言: 询问笔录
4. 物证、书证: 照片、清单等
5. 鉴定意见: DNA、伤情等
6. 勘验、检查、辨认、侦查实验等笔录
7. 电子数据: 监控录像、通话记录
验收标准:
- 自动分类准确率 >85%
- 支持导出证据清单
- 分类维度可调整

### 功能4: 证据矛盾点识别 (P0 - 亮点功能)
优先级: P0
描述: 自动比对各证据之间的矛盾，高亮显示不一致之处，生成矛盾点报告
用户故事: 作为律师，我需要发现证据矛盾，以便质证
实现逻辑:
1. 提取关键事实要素(时间、地点、行为、人物)
2. 交叉对比不同证据的同一要素
3. LLM判断是否矛盾及可信度
4. 生成矛盾清单
验收标准:
- 矛盾点识别准确率 >75%
- 矛盾点覆盖度 >70%
- 可信度评估合理

### 功能5: 质证要点建议 (P0 - 亮点功能)
优先级: P0
描述: 基于证据分类和矛盾分析，自动生成质证要点清单，引用相关法律条文
用户故事: 作为律师，我需要质证要点建议，以便准备庭审
验收标准:
- 质证要点相关性 >80%
- 法律条文引用准确率 >90%
- 用户满意度 >4.0/5.0

## 4. 技术架构

### 4.1 整体架构
```
前端层: React 18 + TypeScript (Web界面) + 移动端适配(响应式)
         ↓
API层: FastAPI (Python 3.11) - 认证、权限、限流、日志
         ↓
业务逻辑层: 文档解析引擎、信息抽取模块、证据分析模块、报告生成模块
         ↓
AI能力层: LLM调用(OpenAI SDK统一接口)、RAG系统(pgvector+知识库)、OCR(PaddleOCR)
         ↓
数据层: PostgreSQL 15 + pgvector(结构化数据+向量检索)、文件存储(MinIO/OSS)
```

### 4.2 核心技术栈
| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | React 18 + TypeScript | 成熟框架，生态完善 |
| 后端 | FastAPI + Python 3.11 | 高性能，AI友好 |
| 数据库 | PostgreSQL 15 + pgvector | 关系型数据库 + 向量检索插件 |
| 向量数据库 | pgvector (MVP阶段) | 轻量级向量检索，无需独立部署 |
| LLM | OpenAI SDK (多模型适配) | 统一接口，支持GPT-4/Claude/DeepSeek等 |
| OCR | PaddleOCR | 本地部署，中文识别准确率高 |
| RAG框架 | LlamaIndex | 专注RAG，性能优异，pgvector原生支持 |
| Embedding | text-embedding-3-small / bge-large-zh | OpenAI API / 本地模型（中文） |
| 部署 | Docker Compose | 容器化，快速部署 |

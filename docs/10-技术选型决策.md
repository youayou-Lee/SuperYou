# 法律AI案件分析产品 - 技术选型

## 一、后端技术栈

### 核心框架
- **语言**：Python 3.11+
- **框架**：FastAPI
- **ORM**：SQLModel

### 主要依赖
```bash
# Web框架
fastapi
uvicorn[standard]

# 数据库
sqlmodel
sqlalchemy
asyncpg           # 异步PostgreSQL驱动
redis

# AI集成
openai            # OpenAI SDK（兼容vllm）
llama-index       # RAG框架

# OCR与文档处理
paddleocr
paddlepaddle
python-docx
pypdf
openpyxl

# 任务队列
arq

# 向量数据库
pgvector          # PostgreSQL向量扩展

# 工具库
pydantic
python-multipart
python-jose[cryptography]
passlib[bcrypt]
```

## 三、数据库方案

- **PostgreSQL 16+**：主数据库
  - 存储所有数据（结构化 + 非结构化JSONB + 向量）
  - 扩展：pgvector（向量检索）

- **Redis 7+**：缓存数据库
  - 任务队列、会话管理、缓存

---

## 四、AI模型方案

### LLM（大语言模型）
- **本地部署**：Qwen2.5-7B-Instruct 或 Qwen2.5-14B-Instruct（vllm推理）
- **API备用**：通义千问 / DeepSeek / Kimi
- **切换方式**：通过环境变量配置，统一使用OpenAI SDK接口

### Embedding（向量化）
- **本地部署**：bge-m3（1024维）
- **API备用**：通义千问Embedding API
- **向量存储**：PostgreSQL + pgvector

### OCR（文字识别）
- **引擎**：PaddleOCR
- **部署**：直接集成到FastAPI服务
- **能力**：文本识别、表格识别、版面分析

### RAG（检索增强）
- **框架**：LlamaIndex
- **场景**：案件语义分析、法律知识增强

## 六、系统架构

```
Vue 3 前端 (Ant Design Vue + ECharts)
         ↓ HTTP/WebSocket
FastAPI 后端 (集成 PaddleOCR + LlamaIndex)
         ↓
    ┌────┴────┬──────────┐
    ▼         ▼          ▼
  vllm     Embedding  PostgreSQL + pgvector
  Qwen     bge-m3     (所有数据)
    │         │            │
    └─────────┴────────────┘
              ↓
          Redis (缓存)
```

## 七、技术决策总结

| 技术领域 | 选择 | 说明 |
|---------|------|------|
| 后端 | FastAPI + Python | AI生态成熟 |
| 前端 | Vue 3 + TypeScript | 开发效率高 |
| UI库 | Ant Design Vue | 企业级组件 |
| 可视化 | ECharts | 强大图表库 |
| 主数据库 | PostgreSQL + JSONB | 统一存储所有数据 |
| 缓存 | Redis | 高性能 |
| 向量库 | pgvector | PostgreSQL扩展 |
| ORM | SQLModel | 轻量级 |
| 任务队列 | Arq | FastAPI集成 |
| OCR | PaddleOCR | 中文识别准确 |
| RAG | LlamaIndex | 成熟框架 |
| Embedding | bge-m3（本地）+ 通义千问（API） | 混合方案 |
| LLM | 本地Qwen（主）+ API（备） | 数据安全 |

---

## 八、核心优势

1. **架构简化**：PostgreSQL统一存储，仅用2个数据库
2. **本地优先**：主要AI模型本地部署，数据安全
3. **成本可控**：使用免费开源模型
4. **灵活切换**：本地/API可配置化切换
5. **中文优化**：PaddleOCR + Qwen + bge-m3

# 刑事案件AI分析产品 - 产品需求文档

## 一、核心痛点

| 痛点 | 具体表现 |
|--------|----------|
| **信息定位难** | 材料太多，难以快速找到关键信息（如罪名、证据、时间线） |
| **分析耗时长** | 人工阅读分析需要数天甚至数周 |
| **遗漏风险** | 容易忽略对辩护有利的细节或矛盾点 |
| **证据分散** | 证据分散在不同材料中，构建犯罪证据链麻烦且耗时 |

---

## 二、核心功能需求

### 模块1：材料上传与智能阅读

**功能描述：**
- 支持批量上传PDF、图片文件
- 自动识别文档类型（立案决定书、起诉意见书、讯问笔录、证据材料等）
- OCR识别（扫描件、图片）
- 文档结构化解析
- 关键信息提取（人名、时间、地点、罪名等）

**验收标准：**
- 支持单次上传100+文件
- OCR准确率>95%（正常扫描件）
- 文档分类准确率>90%

### 模块2：案件基本信息摘要

**功能描述：**
- 自动提取案件要素：
  - 案件编号、案由
  - 被告人、被害人基本信息
  - 涉嫌罪名
  - 立案时间、移送时间
  - 证据数量概览
- 生成案件概览报告（1页可视化）

**验收标准：**
- 要素提取准确率>85%
- 报告生成时间<30秒（100页材料）

### 模块3：时间线自动梳理 ⭐核心功能

**功能描述：**
- 从材料中提取所有时间相关事件
- 按时间顺序自动排列
- 识别关键节点（立案、逮捕、讯问、移送等）
- 生成可视化时间线
- 支持人工标注和调整

**验收标准：**
- 时间提取准确率>90%
- 时间线可视化清晰易读
- 支持事件筛选（按类型、当事人等）

### 模块4：证据链自动构建 ⭐核心功能

**功能描述：**
- 自动识别所有证据（物证、书证、言词证据）
- 提取证据关联关系（A证明B，B佐证C）
- 构建证据链可视化图
- 识别证据链断裂点
- 评估证据链完整性

**验收标准：**
- 证据识别率>90%
- 关联关系准确率>80%
- 可视化清晰展示证据链

### 模块5：矛盾点/疑点智能挖掘 ⭐核心功能

**功能描述：**
- 跨材料自动比对发现矛盾：
  - 供述前后不一致
  - 证据与供述冲突
  - 时间线矛盾
  - 证据间的矛盾
- 生成矛盾点清单
- 标注矛盾严重程度（关键/次要）
- 关联到具体材料页码和内容

**验收标准：**
- 矛盾点发现率>70%（律师标注的矛盾点）
- 误报率<30%
- 定位准确到具体页码

### 模块6：案例检索与对比

**功能描述：**
- 基于案情要素自动检索类案
- 关联北大法宝或威科先行的权威案例库（通过API或爬取）
- 生成类案对比报告
- 量刑建议参考
- 判决要旨提炼

**验收标准：**
- 检索响应时间<5秒
- 类案相关度Top-3准确率>70%
- 提供权威案例链接

### 模块7：辩护思路建议 ⭐核心功能

**功能描述：**
- 基于分析结果生成辩护策略建议：
  - 罪名辩护方向（无罪、轻罪、此罪与彼罪）
  - 证据质证重点
  - 量刑从轻建议
  - 程序性抗辩要点
- 生成辩护策略大纲
- 关联支持法条和类案

**验收标准：**
- 建议合理性>75%（法律专家评估）
- 生成时间<60秒
- 提供可追溯的法律依据

### 模块8：智能问答

**功能描述：**
- 支持自然语言提问
- 基于已上传材料回答案件相关问题
- 引用具体材料内容（页码、段落）
- 支持多轮对话
- 问题历史记录

**验收标准：**
- 响应时间<3秒
- 引用准确率>90%
- 支持上下文理解
